# TANIK — measured performance (skeleton, awaiting Phase 3 #43)

This document will report measured performance of the iris matcher, the fingerprint matcher, and the fused unified-verify decision against held-out test sets. **It is currently a skeleton.** Every number below reads "TBD — awaiting Phase 3 #43 evaluation run." When the harness in `tests/evaluation/` runs against real data, this file is written by the harness, not by hand. **No number in this file has ever been hand-typed by a human or by Claude** — that would violate the honesty discipline (CLAUDE.md, README §"Honest metrics").

> **Status:** skeleton. Will be populated by `tests/evaluation/run_evaluation.py` (`#43`) once `#11` (ND-IRIS-0405) and the FVC-style fingerprint dataset land.

## Reproducibility

The `#43` harness will publish:

- The exact dataset splits (file lists with deterministic ordering).
- The pairing strategy for genuine and impostor pairs.
- The threshold sweep used to compute the ROC curves.
- The git commit SHA at which the numbers were generated.
- The hardware (CPU model, RAM) and software (Python, OpenJDK, OS) versions.

All of the above lives at the top of this file once `#43` writes it, so a reader can rerun and verify.

## Where the calibration_status flips

When `#43` ships:

- The unified-verify response's `calibration_status` flips from `"placeholder"` to `"calibrated"` (env-flagged via `TANIK_FUSION_CALIBRATION_STATUS`, see BACKLOG entry).
- This document is overwritten with measured numbers.
- `docs/fusion.md` is updated to point at the calibration set composition and the resulting weights.
- The README's status line changes from "measured FAR/FRR pending dataset acquisition" to a one-line summary of the headline numbers.

## What this document does NOT yet contain

- Per-attack-type APCER/BPCER for the PAD module (Phase 4; lands when `docs/pad.md` is implemented).
- Cross-dataset generalisation numbers (training on one dataset, evaluating on another). Out of scope for `#43`; useful as a Phase 5 robustness study.
- Fairness analysis across demographic subgroups (age, ethnicity, eye colour). The available iris datasets are demographically narrow and a rigorous fairness analysis would require additional dataset acquisition. Captured as a known unknown for outreach.
